MODULE 1.1 — Grade Sheet Auto Import (Week 1)
📊
Level Register (All Layers)
Source: Original Grade Sheet .xlsx (TBRP format — finalized)
One upload endpoint handles all sheets:
BC, DBM, WMM, CTB, GSB, SUBGRADE, EMBANKMENT TOP

Auto-detection: Layer code from sheet name/title
Auto-mapping: CH column → offsets → SE%/CAMBER% → FRL LME/RME
Flexible JSONB: offset_widths + rl_values stores any project's offset configuration

TBRP ready: 24,584 records (7 layers × 1,756 chainages × L/R)
Table: level_register
Endpoint: POST /reference-data/upload-grade-sheet
Week 1
🌍
OGL Data
Source: OGL sheet in same Grade Sheet file
Structure: Chainage + RL_L30...RL_CL...RL_R30
Full corridor survey — 31 measurement points per chainage every 2m from -30m to +30m

Key calculations enabled by OGL:
Cut/Fill = EMB FRL − OGL RL_CL
Earthwork Volume = Avg area × 10m (CUM)
EMB height validation per chainage

TBRP ready: 3,492 records (1,746 chainages × L/R)
Table: ogl · Endpoint: POST /ogl/upload
Note: OGL RL is authoritative — GPS altitude is location reference only

MODULE 1.5 — Validation Engine + Enriched Approval (Week 4–5)
🔍
Validation Engine
Endpoint: POST /validate/capture/{entry_id}
Auto-runs 3 checks on every pending entry:

Check 1 — Level Register:
Chainage exists in level_register for this layer?
quantity_lm within chainage range distance?

Check 2 — BOQ Balance:
activity_code+stage exists in BOQ?
contracted_qty − approved_qty − this_entry_qty ≥ 0?
"2,500 LM contracted. 1,800 approved. +120 this entry. 580 balance."

Check 3 — Layer Sequence:
stage+layer_section matches TCS layer_stack sequence?
Previous layer approved before this layer?

Returns: ValidationSummary JSON with ✅/⚠️/❌ per check
